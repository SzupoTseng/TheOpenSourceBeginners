# 開源器識 · The Open Source Beginners

> **2026 年最紅、最多人用、最新熱度的 196 個開源專案深度解剖——一本從新手到架構師的選型聖經。**

在 GitHub Trending 像消防栓一樣噴資訊的年代，「哪個最好」是個假問題。這本書把當前最受矚目的 **196 個開源專案**，一個一個攤開，用同一副手術刀解剖，讓你看清每一個專案**紅的真正原因、技術核心硬在哪、以及它哪裡會在半夜咬你一口**。

> 💡 **全書的核心命題**
> **選型的盡頭，不是「用最紅的」，而是「看懂它為什麼紅、以及它哪裡會咬你」。**

## 三個面向，一張表看懂

| 面向 | 符號 | 量的是 | 你買到的 | 最大的雷 |
|------|------|-------|---------|---------|
| **最紅面向** | 🏆 | 聲量、心智佔有率 | 生態與招聘 | 聲量泡沫、紅利已被吃掉 |
| **最多人用** | 👥 | 部署密度 | 確定性與穩定 | 無聊、原理被低估 |
| **最新熱度** | 🔥 | 變化速度 | 未來的期權 | 不穩定、會斷更 |

## 這本書給誰

- 要做技術選型、卻被熱度淹沒的工程師。
- 想補底層原理的轉職者與新進工程師——每個專案都有 **「新人須知（大廠第一週）」**：你會在哪撞見它、最少要會什麼、最常踩的雷。
- 評估開源供應鏈與授權風險的架構師與技術主管。

## 兩點誠實聲明

1. **關於 star 數與維護者數**：每個專案附有「GitHub 體檢」，數字為撰寫時的**量級估計**（以「約」標示），非即時值；只為讓你分辨「這是十萬 star 的地基，還是三千 star 的新芽」。需要精確當日數字，GitHub API 一行即可刷新。
2. **關於長尾與存疑專案**：素材蒐集自一份公開技術搜尋資料，其尾端含少數冷門或難以查證的名稱。本書照實收錄，但明確標記為「新興／待證實」，且**不為其編造 star 數或技術細節**，選型前請務必自行確認上游真實性與活躍度。

## 立場

書中所有「老手視角」的點評，出自一個**中性的資深工程師**視角，不綁定任何公司或職級。我們在乎的從來不是「某大廠怎麼說」，而是「一個踩過夠多坑的人，會怎麼替你把關」。
